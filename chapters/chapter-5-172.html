Since data emits a JSON object, the flatMap operator allows to issue a HTTP request with the web client once that JSON object has been emitted. We need another (nested) flatMap after the HTTP request to the snapshot service has succeeded. Indeed, rxSendJsonObject gives a single observable that emits a HTTP response. However, we need the JSON object because it has to be returned to the requester after sending to the snapshot service has succeeded, so the second flatMap allows to do that and re-inject it in the pipeline. This is a very common pattern with RxJava.
