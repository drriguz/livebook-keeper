Vert.x futures are also inter-operable with the CompletionStage interface of the java.util.concurrent package in the JDK. The CompletionStage interface represents a step in an asynchronous operation, so you can think of it as being a future, especially as there is a class called CompletableFuture that implements CompletionStage. For instance the HTTP client API in Java 11 offers sendAsync methods that return CompletableFuture to make asynchronous HTTP requests.
